Quickstart

Run it locally

From clone to a working tutoring session in about 30 minutes. Seven steps. The first time you do it the Supabase setup takes most of the time; subsequent runs are seconds.

Prerequisites

  • Python 3.11 or newer
  • A free Supabase account (GitHub SSO works)
  • An Anthropic API key (Claude is the default model)
  • A terminal and a browser

The seven steps

  1. Clone and install

    Clone the repo and install the Python dependencies. We recommend a virtual environment.

    git clone https://github.com/kriziadelgado88/OPAS.git
cd OPAS/agent
python -m venv .venv

# macOS / Linux
source .venv/bin/activate
# Windows
.venv\Scripts\activate

pip install -r requirements.txt

  2. Provision Supabase

    Follow the runbook at deployment/supabase/runbook.md. About 15 minutes — covers creating the project, copying credentials, applying the schema, and seeding the reference skill.

    By the end you should have four values to paste into your .env:

    • SUPABASE_URL
    • SUPABASE_ANON_KEY
    • SUPABASE_SERVICE_ROLE_KEY
    • SUPABASE_DB_PASSWORD

  3. Configure environment

    Copy the template and fill in your keys.

    cp .env.example .env
# open .env in your editor and paste the four Supabase values + your Anthropic key
    Do not commit .env. It's gitignored at the repo root, but double-check before pushing if you've added it manually.

  4. Mint a learner token

    The student client requires a bearer token. Mint one for yourself.

    # from agent/
python scripts/mint_learner_tokens.py --email you@example.com

    The script prints a token. Copy it — you'll paste it into the URL in step 7.

  5. Start the agent runtime

    # from agent/
uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload

    Confirm it's up: http://localhost:8001/health should return a JSON response with "env": "dev".

  6. Serve the frontend

    In a second terminal, from the repo root:

    cd app
python -m http.server 8080

  7. Open the demo

    The student client expects a token= query param the first time:

    http://localhost:8080/opas-student.html
  ?skill_id=hks.api318.unit1.thinking-probabilistically
  &token=<your-token>

    The token is cached to localStorage so subsequent loads don't need it.

    Demo mode (live YAML highlights — useful when showing how the protocol works):

    http://localhost:8080/opas-student.html
  ?skill_id=hks.api318.unit1.thinking-probabilistically
  &demo=1

    Wizard (skill authoring): http://localhost:8080/opas-wizard.html

What to test

Once you're in a session, the things to look for:

  • Session starts with a Socratic question (not a lecture)
  • In demo mode: YAML pane populates on load; lines highlight after each turn
  • Citations appear in agent replies ([source, §section] format)
  • Off-topic questions get a polite refusal (the refuse_if_ungrounded rule fires)
  • "Just give me the answer" — agent stays Socratic
  • End session button works and writes session-end events to xapi_events in Supabase
  • Wizard can author a new skill and publish it (you should see it appear in skills table with status='published')

Troubleshooting

"Missing required env var: SUPABASE_URL"

Your .env isn't being read. Confirm it's at agent/.env (not agent/app/.env).

Health check returns 500

Usually means an env var is set but invalid. Check the uvicorn console for the specific error.

Wizard can save drafts but publishing fails

The row-level-security policies require an authenticated educator role. Make sure you ran the educator promotion step in the Supabase runbook.

Student page returns 401

Your bearer token is missing or expired. Mint a new one with mint_learner_tokens.py.

Agent replies are slow / time out

Anthropic API calls can take 5-30 seconds for long replies. If they're consistently timing out, check your API key is valid and your account has credit.
